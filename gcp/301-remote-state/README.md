# GCP Remote State Terraform

A minimal **Terraform** configuration that stores state in a Cloud Storage bucket. Bootstrap scripts create the backend bucket in GCP; Terraform reads the backend settings directly from `backend.tf`.

This example uses Terraform's native remote state backend. It is **not** a Terragrunt setup â€” there is no `terragrunt.hcl`, no generated backend config, and no `terragrunt` commands.

## Purpose

This example builds on [101-basic](../101-basic/). It shows:

- How to configure the native Terraform `gcs` remote state backend in `backend.tf`
- How to bootstrap backend storage with Google Cloud CLI scripts before running Terraform
- How Terraform manages infrastructure when state is stored remotely

The state backend (Cloud Storage bucket) is created outside Terraform by the bootstrap scripts. Terraform itself only manages the sample workload bucket.

## Prerequisites

- [tenv](https://tofuutils.github.io/tenv/) for Terraform version management â€” see [tenv.md](../../tenv.md)
- Terraform `>= 1.15.0` (pinned to **1.15.7** via `.terraform-version`)
- [Google provider](https://registry.terraform.io/providers/hashicorp/google/latest) `~> 5.0`
- [Google Cloud CLI](https://cloud.google.com/sdk/docs/install) installed and authenticated

Install and select the pinned Terraform version:

```bash
tenv tf install    # reads .terraform-version
terraform version  # should report 1.15.7
```

Authenticate before running the bootstrap scripts or Terraform:

```bash
gcloud auth login
gcloud auth application-default login
```

## What gets created

### By bootstrap scripts (remote state backend)

| Resource | Name (default) | Purpose |
|----------|----------------|---------|
| Cloud Storage bucket | `learn-terraform-state-{project-id}-` + 9 random digits | Stores Terraform state objects |

The bootstrap scripts ask which GCP project to use, generate a globally unique bucket name, update `bucket` in `backend.tf`, and create the matching bucket with versioning enabled.

GCS backends use built-in state locking â€” no separate lock table is required.

### By Terraform (workload)

| Resource | Name (default) | Location (default) |
|----------|----------------|--------------------|
| Cloud Storage bucket | `{project-id}-learn-terraform-workload` | `europe-north1` |

## Variables

| Name | Type | Default | Description |
|------|------|---------|-------------|
| `location` | `string` | `europe-north1` | GCP location for the workload bucket |
| `bucket_name_suffix` | `string` | `learn-terraform-workload` | Suffix appended to the project ID for the workload bucket name |

## Outputs

| Name | Description |
|------|-------------|
| `bucket_name` | Name of the workload Cloud Storage bucket |
| `bucket_url` | URL of the workload Cloud Storage bucket |

## Usage

### Step 1 â€” Bootstrap the Cloud Storage backend

Run one of the bootstrap scripts before `terraform init`. The script:

1. Lists your GCP projects and asks which one to use
2. Generates a unique state bucket name like `learn-terraform-state-my-project-342254543`
3. Updates `bucket` in `backend.tf`
4. Creates the bucket with versioning and public access prevention

**Bash:**

```bash
cd gcp/301-remote-state
./bootstrap-backend.sh
```

**PowerShell:**

```powershell
cd gcp/301-remote-state
./bootstrap-backend.ps1
```

Example prompt:

```text
Available GCP projects:

  1) my-project-123
     My Project

  2) sandbox-456
     Sandbox

Select project [1-2]: 2

Using project: sandbox-456

Generated state bucket name: learn-terraform-state-sandbox-456-342254543
Updated bucket in backend.tf
```

Other backend settings (`prefix`) come from `backend.tf`. Only `bucket` is generated by the bootstrap scripts.

### Step 2 â€” Run Terraform

```bash
terraform init
terraform plan
terraform apply
```

Use the `terraform` CLI directly â€” not `terragrunt`. Terraform stores its state in the Cloud Storage bucket configured in `backend.tf`.

### Cleanup

Remove workload resources first:

```bash
terraform destroy
```

Then delete the backend bucket manually:

```bash
gcloud storage rm --recursive gs://YOUR-STATE-BUCKET-NAME
```

Replace the bucket name with the value from `backend.tf`.

## Authentication

The bootstrap scripts and Terraform use Application Default Credentials. Run `gcloud auth application-default login` before `terraform init`.

For CI/CD, use a service account with Storage permissions. See [GCS backend authentication](https://developer.hashicorp.com/terraform/language/settings/backends/gcs).
